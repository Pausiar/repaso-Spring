🌱 1 Introducció a Spring Boot ›
FrameworkJavaMaven/Gradle

Spring Boot és un framework Java basat en Spring Framework que simplifica el desenvolupament d'aplicacions modernes. Redueix la configuració manual i agilitza el treball.

Aplicacions que es poden crear

  • APIs REST: Comunicació entre aplicacions.
  • Aplicacions web: Interfícies d'usuari (Thymeleaf).
  • Microserveis: Aplicacions distribuïdes (Spring Cloud).
  • Apps empresarials: Persistència i seguretat complexa.

Característiques principals

  • ✅ Servidor integrat: Tomcat per defecte → arranca al port 8080.
  • ✅ Auto-configuration: Detecta i configura dependències automàticament.
  • ✅ Dependències amb Maven/Gradle: pom.xml o build.gradle.
  • ✅ Ecosistema complet: Security, Data, Cloud, etc.

Classe principal d'arrencada

@SpringBootApplication public class MyApp { public static void main(String[] args) { SpringApplication.run(MyApp.class, args); } }

L'Ecosistema Spring

  • Spring Framework: IoC, DI, AOP – el nucli.
  • Spring Boot: Facilita la creació ràpida (servidor integrat, anotacions).
  • Spring Data: Persistència amb JPA/Hibernate.
  • Spring Security: Autenticació i autorització.
  • Spring Cloud: Microserveis i aplicacions distribuïdes.
💡 AOP (Aspectes) = afegir crosstutting concerns (logs, transaccions) sense modificar el codi principal.

🔄 3 Arquitectura MVC ›
MVCThymeleafArquitectura

Components del patró MVC

  • 🗃️ Model: Dades + lògica de negoci. Entitats, serveis.
  • 🖼️ Vista: Interfície d'usuari. Plantilles Thymeleaf (.html).
  • 🕹️ Controlador: Gestiona peticions HTTP. Intermediari entre Model i Vista.

Objectius del MVC

  • ✅ Separació de responsabilitats: Cada component té un paper clar.
  • ✅ Reutilització: El Model és reutilitzable.
  • ✅ Escalabilitat i manteniment.

Arquitectura en capes de Spring Boot

HTTP Request ↓ [ Controller ] ← @RestController / @Controller ↓ [ Service ] ← @Service (lògica de negoci) ↓ [ Repository ] ← @Repository (accés a BD) ↓ [ Database ]

Flux d'una petició

1. Client fa GET /ciutats → 2. Controlador rep la petició → 3. Crida al Servei → 4. Servei crida el Repositori → 5. Repositori retorna dades → 6. Controlador retorna JSON o renderitza Vista.

💡 @Controller retorna vistes HTML (Thymeleaf). @RestController retorna JSON/XML.
🔌 4 IoC i Injecció de Dependències (DI) ›
IoCDIBeans

Inversió de Control (IoC)

El contenidor IoC de Spring crea i gestiona els objectes (beans). No som nosaltres qui els creem amb new.

Cicle del contenidor IoC

  1. Inici amb SpringApplication.run()
  2. Escaneja classes amb @Component, @Service, @Repository, @Controller
  3. Crea una instància (bean) per cada una
  4. Injecta les dependències (@Autowired)
  5. Gestiona el cicle de vida del bean

Anotacions principals

  • @Component → Component genèric.
  • @Service → Capa de lògica de negoci.
  • @Repository → Capa d'accés a dades.
  • @Controller / @RestController → Capa de presentació.
  • @Autowired → Injecta una dependència automàticament.
  • @PostConstruct → S'executa DESPRÉS d'inicialitzar el bean.
  • @PreDestroy → S'executa ABANS de destruir el bean.

Exemple: Sense IoC vs Amb IoC

// ❌ SENSE IoC – acoblament fort public class Servei { private Repositori repositori = new Repositori(); // manual! } // ✅ AMB IoC – Spring injecta automàticament @Service public class Servei { @Autowired private Repositori repositori; // Spring ho gestiona }
💡 Avantatges IoC: menys coupling, fàcil de testar (mocks), codi més mantenible.
🗃️ 5 Entitats JPA ›
JPAHibernate@Entity

Anotacions bàsiques d'entitat

  • @Entity → Marca la classe com a taula de BD.
  • @Table(name="nom") → Especifica el nom de la taula.
  • @Id → Clau primària.
  • @GeneratedValue(strategy = GenerationType.IDENTITY) → Autoincrement.
  • @Column(name="nom", nullable=false, length=100, unique=true) → Configuració de columna.
  • @Transient → Camp que NO es guarda a la BD.

Exemple d'entitat

@Entity @Table(name = "usuaris") public class Usuari { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @Column(nullable = false, length = 100) private String nom; @Column(unique = true) private String email; @Transient private String dadesTemporals; // NO es guarda }

Relacions entre entitats

  • @OneToOne → 1 persona – 1 passaport.
  • @OneToMany / @ManyToOne → 1 autor – molts llibres.
  • @ManyToMany → Molts estudiants – moltes assignatures → demana @JoinTable.
  • @JoinColumn(name="fk_col") → Defineix la clau forana.
  • mappedBy → Evita duplicar la relació al costat invers.
  • fetch = FetchType.LAZY → Carrega dades el·lazily (sota demanda).
  • fetch = FetchType.EAGER → Carrega dades immediatament.
💡 JPA no és una implementació, és una especificació. Hibernate l'implementa per defecte a Spring Boot.
📦 6 Repositoris ›
JpaRepositoryCRUD@Query

Creació d'un repositori

@Repository public interface CiutatRepository extends JpaRepository<Ciutat, Long> { // JpaRepository<ENTITAT, TIPUS_CLAU> // Primer paràmetre = tipus entitat // Segon paràmetre = tipus clau primària }

Mètodes CRUD predefinits

  • save(entity) → Guardar/actualitzar.
  • findById(id) → Retorna Optional<T>.
  • findAll() → Retorna List<T>.
  • deleteById(id) → Elimina per id.
  • existsById(id) → Comprova si existeix.
  • count() → Nombre total d'entitats.
  • flush() → Commit forçat a la BD.

Convenció de noms

  • findByNom(String nom) → WHERE nom = ?
  • findByPoblacioGreaterThan(int x) → WHERE poblacio > ?
  • findByNomContaining(String s) → WHERE nom LIKE %s%
  • findByNomAndPais(String n, String p) → AND
  • countByPais(String p) → COUNT WHERE pais = ?
  • deleteByNom(String n) → DELETE WHERE nom = ?
  • existsByEmail(String e) → EXISTS WHERE email = ?

Consulta personalitzada amb @Query

@Query("SELECT c FROM Ciutat c WHERE c.pais = :pais") List<Ciutat> findCiutatsByPais(@Param("pais") String pais); // SQL natiu: @Query(value="SELECT * FROM ciutat WHERE pais=?", nativeQuery=true) List<Ciutat> findByPaisNatiu(String pais);
⚙️ 7 Serveis ›
@ServiceLògica negoci@Autowired

Que és un servei?

Classe que encapsula la lògica de negoci. Intermediari entre Controller i Repository.

Anotacions dels serveis

  • @Service → Registra la classe com a bean de negoci.
  • @Autowired → Injecta repositoris o altres serveis.
  • @PostConstruct → Mètode que s'executa després d'inicialitzar.
  • @PreDestroy → Mètode que s'executa abans de destruir el bean.
  • @Transactional → Gestiona transaccions de BD.

Exemple de servei complet

@Service public class PaisService { private final PaisRepository paisRepository; @Autowired public PaisService(PaisRepository r) { this.paisRepository = r; } public List<Pais> obtenirTots() { return paisRepository.findAll(); } @PostConstruct public void init() { System.out.println("Inicialitzat amb " + paisRepository.count() + " països"); } @PreDestroy public void cleanup() { paisRepository.flush(); } }
🕹️ 8 Controladors ›
@RestControllerHTTPResponseEntity

Tipus de controladors

  • @Controller → Retorna vistes HTML (Thymeleaf). Afegeix dades amb Model.
  • @RestController → Retorna JSON/XML. Combina @Controller + @ResponseBody.

Anotacions de rutes

  • @RequestMapping("/ruta") → Ruta base de la classe.
  • @GetMapping("/ruta") → Petició GET.
  • @PostMapping("/ruta") → Petició POST.
  • @PutMapping("/ruta") → Petició PUT (actualitzar).
  • @DeleteMapping("/ruta") → Petició DELETE.

Captura de paràmetres

  • @RequestParam String nom → URL: /ciutats?nom=Madrid
  • @PathVariable Long id → URL: /ciutats/5
  • @RequestBody Entitat obj → Cos de la petició (JSON → objecte).

Exemple de controlador REST

@RestController @RequestMapping("/api/ciutat") public class CiutatController { @Autowired private CiutatService service; @GetMapping public List<Ciutat> getAll() { return service.findAll(); } @GetMapping("/{id}") public ResponseEntity<Ciutat> getById(@PathVariable Long id) { return service.findById(id) .map(ResponseEntity::ok) .orElse(ResponseEntity.notFound().build()); } @PostMapping public ResponseEntity<Ciutat> create(@RequestBody Ciutat c) { return ResponseEntity.status(201).body(service.save(c)); } @DeleteMapping("/{id}") public ResponseEntity<Void> delete(@PathVariable Long id) { service.deleteById(id); return ResponseEntity.noContent().build(); } }

Codis HTTP importants

  • 200 OK → Petició correcta.
  • 201 Created → Recurs creat.
  • 204 No Content → OK sense cos de resposta.
  • 400 Bad Request → Dades incorrectes.
  • 404 Not Found → Recurs no trobat.
  • 500 Internal Server Error → Error del servidor.
🧱 9 Patrons Builder i DTO ›
BuilderDTOLombok

Patró Builder

Permet construir objectes complexos pas a pas. Útil quan hi ha molts atributs opcionals.

Els 4 components

  • 🔧 Builder (interfície): Defineix els mètodes de construcció.
  • 🔨 Concrete Builder: Implementa la interfície i construeix l'objecte.
  • 🎬 Director: Coordina l'ordre de construcció.
  • 📦 Product: L'objecte final construït.
// Amb Lombok: @Builder fa tot automàticament @Builder @Data public class Persona { private String nom; private String cognoms; private int edat; private String email; // opcional } // Ús: Persona p = Persona.builder() .nom("Jaume") .cognoms("García") .edat(25) .build();

Patró DTO (Data Transfer Object)

Objecte que transfereix dades entre capes sense exposar l'entitat JPA interna.

Avantatges del DTO

  • ✅ Evita exposar dades sensibles (contrasenyes, IDs interns).
  • ✅ Desacobla la capa de presentació de la BD.
  • ✅ Permet personalitzar el format de les respostes.
// Entitat JPA (no s'exposa directament) @Entity public class Usuari { Long id; String nom; String password; } // DTO (el que retorna el controlador) public class UsuariDTO { String nom; // sense password! } // Conversió Entitat → DTO UsuariDTO dto = new UsuariDTO(usuari.getNom());
🔒 10 Spring Security ›
SecurityBCryptJDBC Auth

Que és Spring Security?

Mòdul de Spring que gestiona autenticació (qui ets) i autorització (què pots fer). Protegeix contra CSRF, injeccions, etc.

Dependència

// build.gradle: implementation 'org.springframework.boot:spring-boot-starter-security' implementation 'org.springframework.boot:spring-boot-starter-validation'

Configuració bàsica (application.properties)

spring.security.user.name=jaume spring.security.user.password=1234 spring.security.user.roles=USER

JDBC Authentication – Taules requerides

CREATE TABLE users ( username VARCHAR(50) NOT NULL PRIMARY KEY, password VARCHAR(500) NOT NULL, enabled BOOLEAN NOT NULL ); CREATE TABLE authorities ( username VARCHAR(50) NOT NULL, authority VARCHAR(50) NOT NULL, CONSTRAINT fk FOREIGN KEY(username) REFERENCES users(username) );

Conceptes clau

  • @EnableWebSecurity → Activa la configuració de seguretat.
  • BCryptPasswordEncoder → Encriptació segura de contrasenyes.
  • users → Taula d'usuaris (username, password, enabled).
  • authorities → Taula de rols (ROLE_USER, ROLE_ADMIN…).
  • CSRF: Cross-Site Request Forgery → protegit per defecte.
💡 La contrasenya generada automàticament apareix a la consola:
Using generated security password: abc123...
⚠️ 11 Gestió d'Excepcions ›
@ExceptionHandler@RestControllerAdviceHTTP Status

Tipus de gestió

  • 🏠 Local: Dins d'un controlador específic (@ExceptionHandler).
  • 🌐 Global: Per a tots els controladors (@ControllerAdvice / @RestControllerAdvice).

Gestió local en REST

@RestController public class CiutatController { @ExceptionHandler(ResourceNotFoundException.class) public ResponseEntity<Map<String,String>> handleNotFound( ResourceNotFoundException ex) { Map<String,String> err = new HashMap<>(); err.put("error", "No trobat"); err.put("message", ex.getMessage()); return ResponseEntity.status(HttpStatus.NOT_FOUND).body(err); } }

Gestió global en REST

@RestControllerAdvice public class GlobalExceptionHandler { @ExceptionHandler(Exception.class) @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR) public Map<String,String> handleAll(Exception ex) { Map<String,String> err = new HashMap<>(); err.put("error", "Error intern"); err.put("message", ex.getMessage()); return err; } }

Gestió global en MVC (Thymeleaf)

@ControllerAdvice public class WebExceptionHandler { @ExceptionHandler(Exception.class) public String handleError(Exception ex, Model model) { model.addAttribute("textError", ex.getMessage()); return "plantillaError"; // plantilla .html } }

Diferència clau

  • @ControllerAdvice → Per a controladors MVC (retorna vistes).
  • @RestControllerAdvice → Per a REST controllers (retorna JSON).
📁 12 Ampliació: Upload de Fitxers ›
AmpliacióMultipartFileResource

Configuració (application.properties)

spring.servlet.multipart.max-file-size=10MB spring.servlet.multipart.max-request-size=10MB

Endpoints

  • GET /upload → Mostra el formulari i llista de fitxers (MVC).
  • POST /upload → Puja un fitxer i guarda a upload-dir/ (MVC).
  • GET /uploads/{filename} → Retorna el fitxer com a recurs HTTP (REST).
  • POST /upload/delete/{filename} → Elimina el fitxer (MVC).

Per que dos controladors?

  • 🖼️ Controlador MVC: Gestiona la vista HTML, el formulari, la redirecció.
  • 📡 Controlador REST: Retorna el fitxer com a Resource, controla el Content-Type.
// Controlador MVC – puja fitxer @PostMapping("/upload") public String pujarFitxer(@RequestParam("file") MultipartFile file) { Path dest = Paths.get("upload-dir").resolve(file.getOriginalFilename()); file.transferTo(dest); return "redirect:/upload"; } // Controlador REST – serveix el fitxer @GetMapping("/uploads/{filename}") public ResponseEntity<Resource> servirFitxer(@PathVariable String filename) { Resource r = new FileSystemResource("upload-dir/" + filename); return ResponseEntity.ok() .header(HttpHeaders.CONTENT_DISPOSITION, "inline; filename=\""+filename+"\"") .body(r); }
💡 Els fitxers es guarden físicament a la carpeta upload-dir/ del sistema de fitxers, NO a la base de dades.
